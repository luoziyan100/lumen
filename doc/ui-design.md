## 1. Lumen 是什么(这决定 UI 的本质)

Lumen 不是 chatbot。和 chatbot 最大的区别:**它产出文件**。

一次研究下来,它会在工作区攒下:`notes/`(精读笔记)、`papers/`(抓取的论文正文)、`drafts/`(综述、对比表、报告)。这些是真正的、可累积的产物,不是聊天气泡里划过去就没的话。

| chatbot | Lumen |
|---|---|
| 你只需看它"说了什么" | 你还要看它"**攒了什么**"——读它写的笔记/草稿,甚至自己改 |
| 一问一答 | 多步研究(搜→读→写→综合,可能几十步)+ 多轮追问 |
| 在线即用 | 可后台自治(甩个深任务出去,关窗口,回来看结果) |
| 工具人 | 有人格(Lumen,一个尊重事实、有批判距离的研究伙伴) |

所以 UI 要让**三样东西各得其所**:

1. **对话**(主轴)——问答、追问。
2. **过程**(它在搜/读/写)——多步研究时要可见,但默认折叠,不淹没对话。
3. **产物**(工作区文件)——Lumen 区别于 chatbot 的核心,必须能看、能读。

次要:PDF 阅读器、后台任务列表。

## 2. 设计原则

- **LUI 优先**:主屏是对话。过程、工作区、任务这些"内部状态"放抽屉/侧栏/可折叠区,**不默认堆在主聊天流,不做成仪表盘**。
- **安静**:研究的本质是读—想—写。界面要安静、克制、不抢注意力。留白、低饱和、动效轻。
- **产物易达**:Lumen 攒下的文件是核心资产,要一两步就能看到、读到。
- **诚实可见**:它在搜什么、读到没读到、证据多硬——可查(展开过程),但不强迫看。

## 3. 设计系统约束(必须遵守 —— Lumen Design System · 青瓷 v2,2026-07-04)

> 唯一实现真源:`packages/ui-client/src/tokens.css`(其头部注释与本节互为镜像)。禁在 token 之外硬编码颜色/阴影。

- **基调**:白瓷面 + 青瓷氛围。三层纵深:氛围层(aura,除空态封面外只在边缘可见)→ 纸面层(`--paper-solid`,对话/文档正文所坐,近实底)→ 卡片层(`--card`,输入卡/弹窗/文件卡,白瓷面 + 发丝线 + 浅投影)。立体感来自层级抬升,不来自重阴影。**文字永远不直接压在动效上。**
- **颜色纪律**:青绿只做「品牌与确认」——主强调 `--ember` 深青瓷 #2F6B4E,成功 `--moss` #3F7D57;**链接/动作 `--indigo` 黛蓝 #3A5E70;错误 `--danger` 赭红 #A8452F;警示 `--warning` 琥珀 #8A6420**(aura blocked 态的 UI 强调随之转琥珀)。语义和谐但必须可分辨:错误要能「叫喊」。
- **对比度红线(WCAG 2.1,底 = 纸面层)**:正文类 ≥ 4.5:1,元数据/占位 ≥ 3:1,图形/大字 ≥ 3:1。改色先跑对比度再合入。
- **字体**:UI = Inter Tight + 思源黑体(Noto Sans SC);**长文阅读正文 = Source Serif 4 + 霞鹜文楷**(楷体只在阅读态与问候等点缀位,不进 UI 控件);元数据/代码/DOI = JetBrains Mono。待办:字体本地打包进 Tauri,运行时不依赖 CDN。
- **圆角**:sm 4 / md 8 / lg 12 / full 9999,不超 16。
- **阴影**:4 档 `--shadow-0..3`(none / 发丝浮起 / 卡片 / 弹窗),墨绿灰低透明。**无 inner shadow、无彩色荧光、无 text-shadow。**
- **动效**:安静、短促、线性偏缓。fast 120ms / base 180ms / slow 280ms。禁 spring bounce、overshoot、3D。Hover 染 `--sand`,不用 scale。aura 常驻但被纸面遮护;闲置降耗是已立项待办。
- **控件层**:一律来自 @cloudflare/kumo(无头核 Base UI,皮肤 = theme-celadon 映射本设计系统),禁止手搓按钮/弹层/下拉。
- **控件形制(统一节奏,2026-07-05)**:所有可点控件走 Kumo 形制——高度 sm 26 / base 36,圆角与描边随 Kumo,hover 染 tint,焦点环 focus-ring。**自绘白名单**(仅此四类,超出即打回):列表项(会话/文件条目)、消息气泡、过程行、图标导航钮(.nav-icon-btn)。「像一个个拼上去」的病根就是控件各定形制——新增控件先问 Kumo 有没有,再问白名单。
- **开屏 = 封面(补充)**:问候与输入卡整体**居中偏上**(通行的 agent 开屏形态);开始对话后输入卡落回底部、纸面层铺开。
- **文案**:简体中文,精确优先于热情。不用 emoji,不用 AI 营销话术。界面词汇用用户语言(「资料 / 产物」,不是 Workfolder/Output)。

## 4. 信息架构

```
Lumen 窗口
├─ 对话区(主,常驻)         ← 与 Lumen 问答;过程行内联可折叠
├─ 工作区(产物:文件树+查看) ← 形态待定:抽屉 / 右侧常驻栏 / 内联卡片
├─ 阅读器(PDF / 长文正文)   ← 从工作区或论文链接打开,可与对话并存
└─ 任务(后台研究任务列表)   ← 入口在角落;多任务/关窗续跑时用
```

默认折叠成一行(它干了什么 + 结果计数)。点开看每一步(工具调用/结果),给"诚实可见"但不强迫。

## 6. 整体形态:三选一(请各出一版,看了再定)

差别只在**产物(工作区)怎么摆**。对话区、过程行、阅读器在三者里一致。

### 形态 A — 纯对话 + 工作区抽屉(最守 LUI)
主屏永远是对话;点右上"工作区"从右侧滑出抽屉(§5.4),不看时收起。
```
┌──────────────────────────┐        ┌────────────┐
│ Lumen · 研究    任务 [工作区]│  →点→  │ 工作区抽屉  │
│ (对话,§5.2)              │        │ 文件树+查看 │
│                          │        │ (覆盖右半 │
│ 问…              [发送]   │        │  或浮层)   │
└──────────────────────────┘        └────────────┘
取舍:主屏最纯净;产物藏一层,要点开才见。
```

### 形态 B — 对话 + 工作区双栏(研究台)
左对话、右工作区常驻。像 IDE/Cursor。最能体现"它在帮你攒东西"。
```
┌───────────────┬──────────────────┐
│ Lumen · 研究  │ 工作区            │
│ (对话)        │ 文件树           │
│               │ ────────────     │
│               │ 文件正文预览     │
│ 问… [发送]    │                  │
└───────────────┴──────────────────┘
取舍:产物一眼可见、适合深研究;牺牲一点 LUI 纯净,窄屏挤。
```

### 形态 C — 对话 + 产物内联卡片(无侧栏)
产物作为对话流里的卡片(论文卡 / 文件卡,可点开)。无侧栏。
```
┌──────────────────────────────────┐
│ Lumen · 研究                      │
│ Lumen: 这几篇值得看              │
│  ┌────────────────────────────┐  │
│  │ 预测编码综述 · Clark 2013   │  │ ← 论文卡
│  │ 行为与脑科学 · 开放全文      │  │
│  │ [读全文] [存笔记]           │  │
│  └────────────────────────────┘  │
│ Lumen: 整理了一张对比表          │
│  ┌────────────────────────────┐  │
│  │ 对比表·编码vs流.md  [点开]  │  │ ← 文件卡
│  └────────────────────────────┘  │
│ 问…                       [发送] │
└──────────────────────────────────┘
取舍:产物嵌在语境里、最"对话原生";但对话流会被卡片撑长,翻历史找产物难。
```

> 倾向:你代码注释里的 LUI 原则指向 **A**;但 Lumen "产出文件"是它的命根子,**B** 让产物常驻、最贴研究台的实感。建议 Claude design 把 **A 和 B 各出一版**(C 作为对话内卡片的细节,可融进 A/B),并排看实际观感再定。

## 7. 组件清单(给设计师)

- 标题栏(窗口标题 + 任务入口 + 工作区入口)
- 对话气泡:user / Lumen / 错误 / 系统提示(四种,视觉区分但克制)
- **过程行**(可折叠:摘要态 + 展开的步骤列表)
- Lumen 长回答的排版(它会写带小标题、列表的分析——需要可读的长文排版)
- 论文卡(标题/作者/期刊/年份/开放全文标记/动作)
- 文件卡 / 文件树节点
- 文件查看器(衬线正文)
- PDF 阅读器(分页 + 选区工具:问 Lumen / 存笔记)
- 任务列表项(状态:跑着/完成/中断/失败 + 步数 + 续跑)
- 输入框(多行、可贴链接、运行中禁用)
- 空状态 / 错误条

## 8. 交互与动效

- 过程行展开/收起:base 180ms,高度展开,线性偏缓。
- 工作区抽屉(形态 A)滑入:slow 280ms。
- Lumen 回答**流式**出现(逐段),不要一次蹦出大块——给"它在思考"的呼吸感(但别做打字机逐字,太慢)。
- 新产物落地(写了文件)时,工作区对应项轻微高亮一下(染 --sand,180ms 褪去),不弹窗。
- 全程无 bounce / overshoot。
